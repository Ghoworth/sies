VPP Value Assessment


1. Introduction

Prior to implementing a VPP, the potential value from flexibility needs to be assessed. This is achieved by modelling the system using methods that closely mimic actual VPP operation. Running the models multiple times with different energy asset sizes also allow the potential sensitivity of value to asset choices to be determined.

2. Energy Flow Model

The basis for any value assessment model is an energy flow model of the energy system to be investigated. Figure 1 shows an example for a green hydrogen system with principal energy (and product) flows shown:

Annual assessment models are typical to allow for seasonal variations in demand and generation potential. The energy flow model is used to determine the cashflow associated with importing from and exporting to the network. Additional real (e.g. Renewable Transport Obligation Certificates) or underlying (e.g. battery degradation 'cost') costs can also be incorporated.

3. VPP Logic Integration

As with an implementaton of a VPP system, the value assessment model needs to update required energy flow 'actions' at each timestep. As for a VPP, these can be rule-based or utilise rolling horizon optimisation (a type of model predictive control) using cost-minimising linear programming models.

a) Rule-Based Logic
The simplest VPP system would incorporate fixed, rule-based control logic. At each timestep, control actions are implemented based, typically, on current energy tariffs, storage states or predicted generation relative to demand. The same logic can be applied to value assessment models.

b)Rolling Horizon Optimisation
Particularly for complex energy systems incorporating flexible tariffs, optimising operation to extract maximum value from available flexibility requires more complex mathematics. Rolling horizon optimisation (as characterised in Figure 2) attempts to minimise energy costs with the most effective use of energy assets to meet the required demand over a time 'horizon'. The 'horizon' is typically between 1 and 3-4 days.

4. Input Data Requirements

The concept of 'garbage in-garbage out' also applies to the characterising data used for value assessment modelling. This should, as closely as possible, mimic that used in an actual VPP. Rolling horizon optimisation, in particular, requires future data to be known or predictable. A sample of typical variable input data is shown in Figure 3.
The following are the primary data requirements:

a) Weather/Generation Output
Weather is primarily used to predict output from renewables generation. Using historical weather or generation data is a useful shortcut, but is a source of error in comparison with the forecast data typically used for VPPs. Historical forecast data is, however, difficult to source.

b) Energy Tariffs
VPPs will typically make use of variable time-of-use energy tariffs as a key source of flexibility value. These are often available the day before (e.g. Octopus Agile) giving between 12 and 36 hours of known tariff data depending on the time of day. Beyond this the tariff can only be estimated, typically with recent historical data.

c) Demand
As for weather, historical data can be used as a shortcut, particularly for energy systems with consistent demand patterns. For VPPs, future demand would typically be assessed based on averaged data or regression models if linked to particular time-related factors or weather conditions. For value assessments, using average data is a convenient way to assess this potential source of knowledge error.

d) Energy Asset Characterisation
For rolling horizon optimisation (RHO) models, solving across multiple horizon timesteps requires a degree of simplification to the energy asset models. This applies to both value assessment and VPP models. The following are potential energy assets that could be used in such an assessment:

  • Wind Solar/Thermal PV and other renewables.
  • Battery Storage.
  • EV's (V2G and EV scheduling).
  • Heat Pumps.
  • Flexible (DSR) and non flexible loads.
  • Thermal Storage
  • Electrical Heating
  • Hydrogen electrolysers
  • Hydrogen storage
  • Compressed Air .
  • Liquid Air Storage.
e) Wind Turbine
Typically modelled as a single power output in kWh or MWh per timestep, ignoring power factors and other electrical details. Where weather data is used for assessment of power output, regression models incorporating wind speed and temperature data can be used if actual data is available. The Python library 'windpowerlib' is also used, which incorporates a library of wind speed-to-power curves. Renewablesninja website can be used to get modelled historical data for specific turbine models directly.

f) PV Solar
Typically modelled as a single power output in kWh or MWh per timestep, ignoring power factors and other electrical details. Where weather data is used for assessment of power output, regression models incorporating temperature, cloud cover, humidity and pressure data can be used if actual data is available. The Python library 'pvlib' is also used, which incorporates a library of PV panel and inverter models. Renewablesninja website can be used to get modelled historical data for specific turbine models directly.

g) Battery
Typical modelled as a single state of charge (SOC) variable in kWh or MWh. Normal or recommended operating ranges for Li-ion batteries (e.g. 10-90% SOC) can be incorporated by reducing the useful capacity. Battery charge and discharge efficiencies can vary with SOC and charge/discharge rate and can be modelled with complex equations. This can add significantly to model computation time and convergence probabilty, so often a single % value for efficiency will be used as a shortcut.

h) Thermal Storage
RHO models require simplification of a thermal model from temperatures to a single 'state-of-charge' variable, typically in kWh or MWh, based on the minimum allowable output temperature and input temperature (return from system or direct water supply).

i) Hydrogen Electrolyser
Similar to batteries, conversion efficiency of electricity to hydrogen is dependent on operating conditions (typically loading (% of maximum output) and electrolyser 'stack' temperature). These can be incorporated in the RHO model as a complex set of equations, but a fixed conversion efficiency (e.g. 57kWh/kg H2) would be more typical for value assessments.

5. Value Assessment Output

Annual Cost / Cashflow The majority of VPP system are driven by economic targets (i.e. cost minimisation). Per-timestep cost output data can be analysed and summed for annual cashflow assessment. Detailed Energy Flow Assessment It can be useful to assess the detailed energy flow data also generated by the value assessment model. This can be used, for example, to assess the contribution of renewables generation to total demand, how much generation is used, stored and exported, and peak import and export levels for network connection sizing. This is particularly useful for sensitivity analysis where different combination of asset sizes are compared. Value Assessment Process Value assessments with different input and output requirements are required at different stages of the VPP development process (see Figure 4).


Level 1: Initial High-Level Assessment

Required Inputs:
  • Current demand profiles (half-hourly/hourly) for electricity, heat and hot water.
  • Existing energy assets (e.g. PV, boilers, heat pumps, storage) with basic sizing information.
  • Future planning impacting demand and new/replacement energy assets.
  • Future energy assets and scenarios to be modelled in consultation with VPP team.
  • Site layout and available space for new energy assets.
  • Current energy tariffs.
  • Generic current and future case tariff and grid service income data.
Assessment Process:
  • Half-hourly/hourly energy and cashflow model.
  • 'Opportunistic': Simple per-timestep rule-based analysis of required energy and cash flows, including prioritising use and storage of RES output, using current and future scenarios. Baselining exercise to filter options prior to 'Optimisation' modelling, or
  • 'Optimisation': VPP-replicating moving horizon optimisation model with demand, weather and tariff prediction inputs.
Outputs:
  • Detailed per-timestep energy and cash flow assessment.
  • Storage utilisation and sizing optimisation assessment.
  • CO2 impact.
  • Scope for additional RES.
Go/No-go Basis:
  • Acceptable payback period on additional energy assets and control systems for VPP implementation at 'Future 1' scenario as a minimum.

Level 2: Detailed Feasibility Assessment

Required Inputs:
  • Detailed electrical data for each existing or proposed energy asset.
  • Detailed dimensional and operational data for each existing energy asset (e.g. battery efficiency, storage heat loss etc.)
  • Detailed input data for identified future assets from Level 1 analysis using generic information from SIES library.
  • Local network specific current and future tariff/service model.
Assessment Process:
  • Local grid economic assessment (e.g. pricing volatility, flexibility service potential) based on future scenarios to generate locally-specific data.
  • Grid impact assessment under current fixed limit conditions.
  • Grid impact assessment under future scenarios with dynamic constraints, asset capacity exceeding constraints with G100 limiting etc.
Outputs:
  • Update and refine future strategies developed at Level 1.
  • Determine need for a detailed Level 3 technical assessment of internal electrical dynamics and grid interaction
Go/No-go Basis:
  • Identification of economic scenarios that do not exceed operating and grid limits.

Level 3: Detailed Technical Assessment (optional pre-implementation assessment)

Required Inputs:
  • Applicable initial operating scenarios
Assessment Process:
  • Test of implementation algorithms
  • Identify extreme operating scenarios
  • Detailed power flow model
  • DSO review
Outputs:
  • Worst case situations for internal and external grid interactions.
  • Identify optimal commercial and operating strategies.
